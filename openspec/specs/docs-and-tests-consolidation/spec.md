## 目的
将后端文档与自动化测试收敛到规范目录。

## 需求

### 需求：文档集中到 backend/docs
系统 应当 将后端架构、迁移与运维文档统一放在 `backend/docs/` 下，并按主题组织子目录。

#### 场景：新增迁移文档
- **当** 贡献者新增迁移或架构文档
- **则** 文件 应当 创建在 `backend/docs/` 下，并从对应索引/README 建立链接

### 需求：自动化测试集中到 backend/tests
系统 应当 以 `backend/tests/` 作为自动化测试的唯一规范目录，并通过标准 pytest 发现机制执行。

#### 场景：执行测试套件
- **当** 运行后端校验测试
- **则** pytest 应当 从 `backend/tests/` 自动发现并执行测试，无需脚本特定入口

### 需求：测试代码与工具脚本分离
系统 应当 不在 `backend/scripts/` 保留生产测试用例；该目录 应当 仅用于工具、运维与非测试自动化脚本。

#### 场景：scripts 目录出现测试文件
- **当** 在 `backend/scripts/` 下新增具有测试意图的文件
- **则** 结构校验 应当 失败，并要求迁移到 `backend/tests/`
