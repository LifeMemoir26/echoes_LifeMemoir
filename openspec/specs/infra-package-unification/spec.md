## 目的
统一基础设施包命名空间与契约边界。

## 需求

### 需求：统一基础设施命名空间
系统 应当 使用 `backend/src/infra/` 作为数据库、LLM 与工具适配器的唯一基础设施命名空间。

#### 场景：导入基础设施模块
- **当** 应用模块导入基础设施依赖
- **则** 导入路径 应当 使用 `src.infra.*`，且 应当 NOT 使用 `src.infrastructure.*`

### 需求：移除重复基础设施目录树
系统 应当 在迁移完成后移除 `backend/src/infrastructure/`，并阻止任何新文件再次写入该路径。

#### 场景：迁移完成校验
- **当** 迁移后运行结构校验
- **则** `backend/src/infrastructure/` 应当 不存在或为空，并被标记为禁止新增路径

### 需求：定义 infra 接口与错误契约
系统 应当 在 `application` 边界定义稳定的适配器契约，覆盖 infra 提供能力的请求/响应结构与错误模型预期（例如超时、限流、持久化失败类别）。

#### 场景：infra 适配器失败传播
- **当** infra 适配器抛出超时或限流错误
- **则** application 层 应当 收到已类型化、文档化的错误类别，以供工作流级处理
